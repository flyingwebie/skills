# CORE-EEAT Benchmark — 80-Item Content Quality Framework

## Overview
CORE-EEAT (Content Optimization for Real Expertise, Experience, Authoritativeness, and Trustworthiness) is an 80-item framework across 8 dimensions for measuring and optimizing content quality for both traditional SEO and AI engine optimization (GEO).

## Scoring System
- Each item scored 0-100 (percentage completion)
- Dimension score = average of 10 items in that dimension
- Overall CORE-EEAT score = weighted average of all 8 dimensions
- Minimum viable score: 60/100
- Target score: 80+/100

## C: Content Quality (10 items)

| # | Item | Criteria | Check |
|---|------|----------|-------|
| C01 | Unique value | Content provides information not found on competing pages | |
| C02 | Direct answers | Key questions answered in first 150 words of relevant section | |
| C03 | Depth | Topic covered comprehensively (no major subtopics missing) | |
| C04 | Readability | Flesch score 50+ (appropriate for audience) | |
| C05 | Structure | Logical H1→H2→H3 hierarchy, scannable sections | |
| C06 | Freshness | Content updated within last 12 months, no stale data | |
| C07 | Media | Relevant images, videos, or diagrams that add value | |
| C08 | Accuracy | All facts, statistics, claims verifiable and correct | |
| C09 | Completeness | User doesn't need to visit another page to complete their task | |
| C10 | Engagement | Content invites action (CTA, comment, share, next step) | |

## O: Optimization (10 items)

| # | Item | Criteria | Check |
|---|------|----------|-------|
| O01 | Title tag | 50-60 chars, primary keyword, compelling | |
| O02 | Meta description | 150-160 chars, includes keyword, clear CTA | |
| O03 | H1 tag | One per page, includes primary keyword | |
| O04 | Keyword placement | Primary keyword in title, H1, first 100 words, URL | |
| O05 | Internal links | Min 3 relevant internal links with descriptive anchors | |
| O06 | Schema markup | Appropriate schema type implemented correctly | |
| O07 | Image optimization | Alt text, compressed, descriptive filenames | |
| O08 | URL structure | Clean, descriptive, includes keyword, no parameters | |
| O09 | Mobile friendly | Responsive, touch targets 44px+, readable without zoom | |
| O10 | Page speed | Core Web Vitals: LCP<2.5s, FID<100ms, CLS<0.1 | |
