Create the router skill that the UX Agent calls to get relevant psychology skills per section.

**`skills/psychology-router/SKILL.md`**:

Frontmatter:
```
---
name: psychology-router
description: >
  Routes page sections to relevant psychology skills. Given a page-type and section-type,
  returns 2-4 focused skills instead of all 14. Called by the UX Agent before analyzing
  each section of a page layout.
---
```

**Purpose**: Prevents the "wall of psychology" anti-pattern. Instead of applying all 14 cognitive psychology skills to every section, the router selects the 2-4 most relevant skills based on section type and page context.

**Usage** (how the UX Agent calls this):
```
Input:
  - page_type: string (e.g., "landing-page", "homepage", "product", "content", "checkout")
  - section_type: string (e.g., "hero", "pricing", "cta", "testimonials", "features")
  - conversion_intent: "high" | "medium" | "low" (optional, defaults to "medium")

Output:
  - skills: string[] (2-4 skill names)
  - rationale: string (why these skills were selected)
  - skill_summaries: object[] (name + core principle + key patterns for each selected skill)
```

**Routing Algorithm**:
1. Look up `section_type` in `references/section-skill-map.md` master table
2. Start with primary skills (always included)
3. If `conversion_intent` is "high" OR page_type has a boost modifier, add secondary skills up to max 4
4. Apply page-type modifier from the modifiers table
5. Cap at 4 skills maximum
6. Load summaries from `references/skill-summaries.md` for selected skills
7. Return skills array + rationale + summaries

**Example**:
```
Input: page_type="landing-page", section_type="hero"
Output:
  skills: ["cognitive-load", "curiosity-gap", "visual-cues-cta-psychology", "loss-aversion-psychology"]
  rationale: "Hero on landing page: cognitive-load ensures clean first impression, curiosity-gap hooks visitor, visual-cues guides to CTA, loss-aversion boosted for high-conversion page type"
```

```
Input: page_type="content", section_type="blog"
Output:
  skills: ["curiosity-gap", "cognitive-fluency-psychology"]
  rationale: "Content page: lighter touch — curiosity-gap for headline engagement, cognitive-fluency for readable body"
```

**When to Load Full Skills**: The summaries are enough for UX brief generation. Only load the full skill file (`~/.claude/skills/{name}/SKILL.md`) when:
- User requests deep psychology analysis
- Section is the primary conversion point of the page
- Debugging why a design recommendation feels off

**Reference Files**:
- `references/section-skill-map.md` — Master routing table and selection rules
- `references/skill-summaries.md` — Quick-reference summaries of all 14 skills
