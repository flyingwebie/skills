  <name>Task 1: Create animation reference guide with library-specific patterns</name>
  <files>skills/animation/references/animation-guide.md</files>
  <action>
Create `skills/animation/references/animation-guide.md` covering the four required animation libraries with concrete, copy-paste-ready patterns.

**Structure the guide into 5 sections:**

**1. Entrance Animations (Framer Motion)**
Cover the most common entrance patterns with exact Framer Motion props:
- Fade up: `initial={{ opacity: 0, y: 20 }}` `animate={{ opacity: 1, y: 0 }}` with duration 0.6s, ease "easeOut"
- Fade in: opacity only, duration 0.4s
- Scale in: from scale 0.95, duration 0.5s
- Slide in from left/right: x offset +/- 40px, duration 0.6s
- Stagger children: `staggerChildren: 0.1` in parent's `transition` variants
- Viewport trigger: `whileInView` with `viewport={{ once: true, margin: "-100px" }}`

For each pattern include: the Framer Motion JSX props (initial, animate, transition), recommended duration range, recommended easing, and when to use it (which section types benefit from this entrance).

Map entrance types to section types:
| Section Type | Recommended Entrance | Why |
|---|---|---|
| hero | fade-up, stagger children | Draws attention to headline first, then supporting elements |
| features | stagger grid items | Progressive reveal reduces cognitive load |
| testimonials | fade-in | Subtle appearance maintains reading flow |
| pricing | scale-in | Slight emphasis signals importance |
| CTA | fade-up with delay | Arrives after surrounding content establishes context |
| footer | none or fade-in | Low priority, minimal distraction |

**2. Scroll Behaviors (GSAP + ScrollTrigger)**
Cover scroll-triggered animations with exact GSAP syntax:
- Parallax backgrounds: `gsap.to(element, { yPercent: -20, scrollTrigger: { trigger, start: "top bottom", end: "bottom top", scrub: true } })`
- Progress reveal: scrub-based opacity/transform tied to scroll position
- Pin sections: `pin: true` with `start: "top top"` and `end: "+=100%"` for scroll-locked storytelling
- Counter animations: `gsap.to(counter, { innerText: targetValue, duration: 1, snap: { innerText: 1 }, scrollTrigger: { trigger, start: "top 80%" } })`

Include ScrollTrigger configuration patterns:
- `start` / `end` values and what they mean
- `scrub: true` vs `scrub: 0.5` (smooth scrub)
- `once: true` for fire-once animations
- `markers: true` for development debugging (note: remove in production)

Map scroll behaviors to section types:
| Section Type | Recommended Scroll Behavior | Configuration |
|---|---|---|
| hero | parallax background image | scrub, yPercent: -30 |
| features | progressive reveal on scroll | start "top 80%", once |
| stats/metrics | counter animation | start "top 80%", once |
| testimonials | fade sequence | stagger on scroll enter |
| portfolio | parallax + reveal | scrub for images |

**3. Page Transitions (Swup.js)**
Cover page transition patterns:
- Fade transition: opacity 0 -> 1 with 300ms duration
- Slide transition: translateX with 400ms duration
- Cross-fade: overlap old and new pages with 500ms duration
- Configuration: `new Swup({ animationSelector: '[class*="transition-"]', plugins: [...] })`
- CSS class pattern: `.transition-fade`, `.is-animating`, `.is-leaving`, `.is-rendering`
- Include the required CSS for each transition type
- Note: Swup handles SPA-like transitions -- recommend for multi-page sites only

**4. Smooth Scroll (Lenis)**
Cover Lenis integration:
- Basic setup: `const lenis = new Lenis({ duration: 1.2, easing: (t) => Math.min(1, 1.001 - Math.pow(2, -10 * t)), orientation: 'vertical', smoothWheel: true })`
- RAF loop integration: `function raf(time) { lenis.raf(time); requestAnimationFrame(raf); }`
- GSAP integration: `lenis.on('scroll', ScrollTrigger.update)` and `gsap.ticker.add((time) => lenis.raf(time * 1000))`
- Anchor scroll: `lenis.scrollTo('#section', { offset: -80, duration: 1.2 })`
- Duration and easing recommendations by content type:
  - Long-form content: duration 1.0-1.2, gentle easing
  - Portfolio/showcase: duration 1.4-1.6, smooth easing
  - E-commerce: duration 0.8-1.0, snappy easing

**5. Psychology-Animation Mapping**
Map psychology skills to animation behaviors:
| Psychology Skill | Animation Recommendation | Rationale |
|---|---|---|
| cognitive-load | Stagger reveals, max 3 simultaneous animations | Prevents overwhelm |
| progressive-disclosure | Scroll-triggered reveals, accordion animations | Controls information flow |
| curiosity-gap | Partial reveals on scroll, peek animations | Teases content below fold |
| visual-cues-cta | Subtle pulse or glow on CTA, delayed entrance | Draws eye without distraction |
| halo-effect | Smooth, premium-feeling transitions (longer duration, refined easing) | Reinforces quality perception |
| trust-psychology | Minimal, predictable animations -- no surprises | Builds confidence |
| social-proof | Counter animations for metrics, card reveals for testimonials | Adds dynamism to proof points |

Include a "Don'ts" section: no animation on body text reading, no infinite loops except loading states, no animations longer than 1s for entrances, no layout shift from animations (use transform/opacity only), respect prefers-reduced-motion media query.

  <name>Task 2: Create animation choreography skill with per-page spec generation</name>
  <files>skills/animation/SKILL.md, SKILL.md</files>
  <action>
Create `skills/animation/SKILL.md` following the exact conventions from existing skills (see `skills/copy-generation/SKILL.md` and `skills/persistence/SKILL.md` for format).

**Frontmatter:**
```yaml
name: animation
description: >
  Generates per-page animation choreography specs. Reads UX briefs for psychology context
  and design tokens for style-appropriate motion. Produces implementation-ready specs with
  Framer Motion, GSAP + ScrollTrigger, Swup.js, and Lenis code patterns.
```

**Interface Contract:**

Input:
| Parameter | Type | Required | Description |
|---|---|---|---|
| page | string | Yes | Page name (must have status "designed" in state.json) |
| uxBrief | file | Yes | UX brief from .ui-ux/briefs/{page}-ux-brief.md |
| tokens | object | Yes | Design tokens from .ui-ux/tokens.json |

Output:
- Animation spec file at `.ui-ux/briefs/{page}-animation-spec.md`
- Updated `state.json.animationSpecs[]` with new entry

**Process (5 steps):**

1. **Pre-Flight:** Read .ui-ux/state.json, verify page exists and has status "designed" or "iterated". Read tokens.json for UI style context. Read .ui-ux/briefs/{page}-ux-brief.md for section structure and psychology skills applied.

2. **Section Analysis:** For each section in the UX brief:
   - Identify the section type (hero, features, pricing, etc.)
   - Read which psychology skills were applied
   - Look up the animation-guide.md for: recommended entrance animation for this section type, recommended scroll behavior, psychology-animation mapping for the applied skills
   - Determine if animations should be more restrained or more expressive based on UI style from tokens (Minimalism = restrained, Dark Mode Premium = more expressive)

3. **Choreography Assembly:** Build the full-page animation choreography:
   - **Entrance sequence:** Order and timing of section entrances as user scrolls down. Define stagger delays between sections (typically 0ms -- each section triggers independently on scroll). Define within-section stagger for child elements (100-150ms between children).
   - **Scroll behaviors:** Which sections get parallax, pinning, or scrub-based effects. Reference GSAP ScrollTrigger config from animation-guide.md with exact start/end values.
   - **Page transitions:** If project has multiple designed pages, recommend Swup.js transition type. Match transition style to UI style (Minimalism = fade, Bold/Vibrant = slide).
   - **Smooth scroll:** Lenis configuration recommendation based on content type. Include GSAP integration snippet.
   - **Reduced motion fallback:** For every animation, specify the prefers-reduced-motion alternative (typically: remove transform animations, keep opacity fades at 50% duration, disable parallax, disable smooth scroll).

4. **Spec Generation:** Write the animation spec to `.ui-ux/briefs/{page}-animation-spec.md` with this structure:
   ```markdown
   # Animation Spec: {Page Name}

   ## Overview
   - UI Style: {from tokens}
   - Animation Intensity: {restrained | moderate | expressive}
   - Smooth Scroll: {Lenis config}
   - Page Transition: {Swup type if multi-page}

   ## Section Choreography

   ### {Section Name}
   **Entrance:** {Framer Motion props -- initial, animate, transition}
   **Scroll:** {GSAP ScrollTrigger config if applicable}
   **Children Stagger:** {staggerChildren value}
   **Reduced Motion:** {fallback behavior}

   [repeat for each section]

   ## Page Transition
   {Swup.js configuration and CSS}

   ## Smooth Scroll Setup
   {Lenis initialization with GSAP integration}

   ## Accessibility
   {prefers-reduced-motion CSS media query and JS detection}
   ```

5. **Persistence:** Add entry to `state.json.animationSpecs[]`:
   ```json
   { "page": "{page}", "specPath": ".ui-ux/briefs/{page}-animation-spec.md" }
   ```
   Update `state.json.updatedAt`.

**UI Style to Animation Intensity Mapping:**
| UI Style | Intensity | Notes |
|---|---|---|
| Minimalism | restrained | Fade-only entrances, no parallax, subtle scroll |
| Clean Modern | restrained | Fade + slight translate, minimal scroll effects |
| Editorial | moderate | Elegant reveals, text-focused animations |
| Bold / Vibrant | expressive | Larger transforms, parallax, energetic timing |
| Organic / Rounded | moderate | Soft, flowing animations, gentle easing |
| Technical / Data-Driven | restrained | Precise, grid-aligned reveals, counter animations |
| Dark Mode Premium | expressive | Dramatic reveals, glow effects, sophisticated scroll |

**Integration with /design command:** Note at the bottom of the skill that the /design command can optionally invoke this skill after UI Agent completes screen generation. The animation spec is a supplementary deliverable, not required for screen generation. It can also be run standalone after design is complete.
