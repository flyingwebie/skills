### Step 1 -- Read Registry

Follow the same delegation pattern as generate.md Step 1:
1. Read `@${CLAUDE_PLUGIN_ROOT}/skills/persistence/SKILL.md`
2. Initialize `.etch-components/` if it does not exist (follow the persistence skill's Initialization flow)
3. Read `registry.json`
4. If the `components` array is empty, display: "No components in the library yet. Use `/generate` to create your first component." and stop.

### Step 2 -- Parse Arguments

Instructions for Claude to parse `$ARGUMENTS` into one of four modes:
1. If `$ARGUMENTS` is empty → mode is **list-all**
2. If `$ARGUMENTS` contains `--paste <name>` → mode is **paste** (handled in Step 5, added by Plan 02)
3. If `$ARGUMENTS` contains `--type <type>` → extract the type value; validate it against the enum (`section`, `card`, `form`, `nav`, `footer`, `hero`, `custom`). If invalid, display: "Invalid type '{value}'. Valid types: section, card, form, nav, footer, hero, custom" and stop. Remaining non-flag text is the search term.
4. If `$ARGUMENTS` contains `--tag <tag>` → extract the tag value. Remaining non-flag text is the search term.
5. If `$ARGUMENTS` is a plain string with no flags → mode is **search**, the entire string is the search term.

Note: search term, --type, and --tag can be combined. When combined, apply intersection logic (all conditions must match).

### Step 3 -- Filter Components

Instructions for filtering the `components` array based on the parsed mode:

**List all:** Use the full `components` array unfiltered.

**Search:** Filter where `name` OR `description` contains the search term as a case-insensitive substring.

**Filter by type:** Filter where `type` equals the provided type value (case-insensitive comparison).

**Filter by tag:** Filter where `tags` array contains an entry matching the provided tag (case-insensitive).

**Combined:** Apply all active filters as an intersection — a component must match ALL provided criteria.

After filtering, if no components match, display: "No components match your search." and stop.
