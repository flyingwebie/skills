**skills/etch-conventions/SKILL.md** — Full conventions skill with YAML frontmatter:
```yaml
---
name: etch-conventions
description: >
  Etch best practices for component generation. Encodes Section > Container hierarchy,
  AutoBEM naming alignment, container-query responsive approach, and accessibility defaults.
  Consult this skill before generating any HTML, CSS, or JS for Etch components.
---
```

Body sections:

1. "## Element Hierarchy" — Document the 15 Etch elements from STACK.md (Section, Container, Div, Heading, Text, Anchor, Image, Dynamic Image, Svg, Iframe, Html, Element, Condition, Loop, Empty) with their HTML tag mappings. Emphasize the core pattern: `Section > Container > [content elements]`.

2. "## AutoBEM Naming Rules" — Encode the rules the plugin's BEM output must follow:
   - Block = component name in kebab-case (e.g., `pricing-table`)
   - Elements = flat, single depth only: `.pricing-table__heading` NOT `.pricing-table__header__heading`
   - Element names map to Etch element types: heading -> `__heading`, text -> `__text`, image -> `__image`, container -> `__container`, cta/anchor -> `__cta`
   - Modifiers use `--`: `.pricing-table--featured`
   - CSS uses `&__` / `&--` stemming (Etch CSS panel expands these)

3. "## Container-Query Responsive Approach" — Document that Etch uses container-query-first, not media queries:
   - Components define their own responsive thresholds based on available space
   - Use `@container` syntax, same as `@media` syntax
   - Parent elements may need `container-type: inline-size` declaration
   - No global breakpoints — each component adapts contextually
   - Note open question: exact container query behavior in Etch may require testing

4. "## Accessibility Defaults" — MUST include all of these (per research pitfall 5):
   - Semantic landmarks: use `<section>`, `<nav>`, `<header>`, `<footer>` instead of generic `<div>`
   - Heading hierarchy: one `<h1>` per page, sequential levels (h2 follows h1, h3 follows h2)
   - Image defaults: `alt` text patterns, `loading="lazy"`, `decoding="async"`
   - Motion: respect `prefers-reduced-motion` in animations
   - Focus: `:focus-visible` outline on all interactive elements
   - ARIA: `aria-expanded`, `aria-hidden`, `aria-label` patterns for interactive components
   - Skip links and landmark roles where appropriate

5. "## Section Structure" — Reference directive: "Read `references/section-patterns.md` for HTML templates of all 7 section types."

6. "## Grid System" — Reference directive: "Read `references/grid-system.md` for grid classes and responsive patterns."

7. "## CSS Output Rules" — Key rules for generated CSS:
   - Use `&__` / `&--` stemming (Etch-compatible)
   - Reference only ACSS custom properties for color, spacing, typography values
   - Component-specific layout (grid, flex, positioning) is the ONLY thing that gets raw CSS values
   - Read `@${CLAUDE_PLUGIN_ROOT}/skills/css-generation/references/acss-tokens.md` for token lookup
